## 我们是谁

**孔明伴生团** 是一个研究多智能体治理的实践团队。我们不卖软件、不做框架——我们自己在跑一个多 Agent 系统，然后把跑出来的经验写下来分享。

---

## 我们发现了什么

当所以都在教人「怎么搭建多 Agent 系统」的时候，没有人回答一个问题：**搭起来之后，怎么管？**

我们在自己的系统里踩了一遍坑，换了一个思路——**不给 AI 规矩，给它一个参照系。**

---

## 参照系：五份文件，一个系统

一个 AI 的行为失控，不是因为它不够聪明——是因为它在每一步都要猜「我这么干行不行」。

参照系解决了这个问题：

| 文件 | 回答的问题 | 它是什么 |
|------|-----------|---------|
| SOUL.md | 我是谁？ | 身份锚点——知道自己是谁，行为自然有方向 |
| CONSTITUTION.md | 边界在哪？ | 边界参照——圈外不碰，圈内自由 |
| AGENTS.md | 怎么干活？ | 工作逻辑——每一步都有判定条件（未来路由的时候，这一部分也是拆分的对象之一） |
| CEO.md | 往哪走？ | 目标对齐——短期规划与北极星 |
| 系统导航.md | 什么时候读什么？ | 信息路由——按需加载，不占脑子 |

**边界之内完全自由，边界之外根本不碰。** 这是「强大」和「稳定」能同时成立的原因。

---

## 这套方法跑过哪些地方

- **Claude Code** — 多 Agent 并行协同（很稳）
- **OpenInterpreter** — 安全性明显提升，能力未打折
- **Codex CLI** — 自主编码但不越界
- **open clawI**—如果你是云端扩展性强的话，建议你解决上下文肿胀的问题
- **我们自己的幕僚团镇星系统** — 已经出现协同进化现象
